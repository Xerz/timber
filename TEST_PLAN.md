# План тестирования (Tauri + macOS)

**Цель**
Проверить корректность логики и UI на macOS и вручную подтвердить Windows‑специфичную часть (реестр, запуск exe).

**Инструменты**
- Node.js 20+.
- `node --test` для unit/интеграционных тестов.
- Playwright (Chromium) для UI‑скриншотов и hover.

**Unit‑тесты (JS, macOS)**
1. Фильтрация игр `enabled && available`.
2. Маппинг карточек (title, image, badges).
3. Правила “Бесплатная”.
4. Приоритет `gamePath/workPath/args` над дефолтами.

**Интеграционные тесты (JS, macOS)**
1. Сборка карточек из фикстур.
2. Отработка отсутствующих полей.

**UI‑тесты (Playwright, macOS)**
1. Рендер плиток из `mock-data.json`.
2. Проверка hover‑стиля.
3. Проверка клика (ставится класс `is-launching`).

**Windows‑специфичные ручные тесты**
1. Чтение реестра `last_server` и `auth_token`.
2. Запуск игры с рабочей директорией и аргументами.
3. Desktop закрывает лаунчер без запуска exe.

**Критерии готовности**
- Unit и UI‑тесты проходят на macOS.
- Ручные тесты на Windows подтверждают запуск и реестр.
